Stack DSA

A stack is LIFO — last in, first out. In JS, an array with push/pop is a stack. It shines whenever you need to remember "the most recent unmatched thing."

Operations

OperationBig-O
PushO(1)
PopO(1)
Peek (top)O(1)
SearchO(n)

Use it for

  • Nested structures — matching brackets, HTML tags, JSON depth.
  • Expression evaluation — postfix (RPN), operator precedence.
  • Backtracking / undo — DFS, browser history, function call stack.
  • Monotonic stack — next greater/smaller element in O(n).

Common questions

  • Valid Parentheses — push openers, pop & match on closers.
  • Min Stack — track the running minimum alongside each push.
  • Next Greater Element — monotonic decreasing stack.
Pattern: "match nested pairs" or "most recent unresolved item" → stack. "next greater/smaller for each element" → monotonic stack.
Gotcha: For Valid Parentheses, an empty stack when you hit a closing bracket means invalid, and a non-empty stack at the end also means invalid (unclosed openers).

🎯 Interview questions

How does a MinStack get the minimum in O(1)?

Keep a parallel stack (or store pairs) where each entry records the min of the stack up to that point. Pop both together; the top of the min-stack is the current minimum.

What is a monotonic stack?

A stack kept in sorted (increasing or decreasing) order by popping elements that violate the order. It answers "next greater/smaller element" for all items in O(n).

Why a stack for Valid Parentheses?

The most recently opened bracket must close first — LIFO exactly models nesting.

Where do stacks show up implicitly?

The call stack for recursion, DFS, and undo/redo or browser back/forward.
