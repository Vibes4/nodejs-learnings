Heap / Priority Queue DSA

A binary heap is a complete tree (stored in an array) where every parent beats its children — a min-heap keeps the smallest at the root, a max-heap the largest. It's the implementation behind a priority queue: always pop the "best" element next.

Complexity

OperationBig-O
Peek (min/max)O(1)
Insert (push)O(log n)
Extract (pop)O(log n)
Build heap from arrayO(n)

Use it for

  • Top-K problems — keep a heap of size k; O(n log k) instead of sorting everything.
  • Scheduling — always run the highest-priority / earliest task next (Dijkstra, task queues).
  • Streaming data — running median (two heaps), kth largest in a stream.

Array layout

For a node at index i: parent is (i-1)/2, children are 2i+1 and 2i+2. Insert appends then "bubbles up"; pop swaps root with the last element then "sinks down".

Common questions

  • K Largest Elements — min-heap of size k.
  • Top K Frequent Elements — count with a map, then heap (or bucket sort) by frequency.
Pattern: "top K" or "K largest/smallest/most-frequent" → maintain a heap of size K. For the K largest, use a min-heap so the smallest of your K is cheap to evict.
Gotcha: A heap is only partially ordered — it gives you the extreme element in O(1), but it does NOT give a fully sorted list without repeatedly popping (which is heap sort, O(n log n)).

🎯 Interview questions

Why a min-heap for K largest?

Keep exactly K elements; the root is the smallest of them. For each new value, if it beats the root, pop the root and push it. The heap ends holding the K largest, in O(n log k).

Heap vs sorting for top-K?

Sorting is O(n log n); a size-K heap is O(n log k). When k ≪ n, the heap wins and uses less memory.

How is a heap stored?

As a flat array; parent/child indices are computed arithmetically — no pointers needed.

How do you get a running median?

Two heaps: a max-heap for the lower half and a min-heap for the upper half, kept balanced. The median is the root(s).
