Queue DSA

A queue is FIFO — first in, first out. It's the engine behind breadth-first search and any "process in arrival order" workload.

Operations

OperationBig-O
Enqueue (add to back)O(1)
Dequeue (remove from front)O(1)
Peek (front)O(1)

Use it for

  • BFS — level-order traversal of trees and graphs, shortest path in unweighted graphs.
  • Scheduling — task queues, the event loop's callback/timer queues, printer jobs.
  • Buffering / streaming — producer/consumer pipelines.

Variants

  • Circular queue — fixed-size ring buffer; front/rear wrap around with modulo. Avoids shifting elements.
  • Deque — double-ended; push/pop at both ends (sliding-window maximum).
  • Queue from two stacks — an in stack and an out stack; amortized O(1) dequeue.
Gotcha: In JS, array.shift() is O(n) because it reindexes. For large queues use a head index, a linked list, or a circular buffer to keep dequeue truly O(1).
Pattern: "shortest path in an unweighted graph" or "process level by level" → BFS with a queue.

🎯 Interview questions

Queue vs stack — which for BFS vs DFS?

BFS uses a queue (FIFO, explores by levels); DFS uses a stack or recursion (LIFO, goes deep first).

How do two stacks make a queue?

Push onto an "in" stack. To dequeue, if "out" is empty, pour all of "in" into "out" (reversing order), then pop "out". Amortized O(1).

Why a circular queue?

A fixed array with wrap-around indices reuses freed slots without shifting, giving true O(1) enqueue/dequeue and bounded memory.

Why avoid Array.shift() for a queue?

It removes the first element and reindexes the rest — O(n) per dequeue.
