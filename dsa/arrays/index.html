Arrays DSA

Arrays are contiguous, index-addressable memory. The interview game is recognizing which of four patterns collapses a brute-force O(n²) into O(n).

Complexity

OperationBig-O
Access by indexO(1)
Search (unsorted)O(n)
Insert / remove at endO(1) amortized
Insert / remove at frontO(n) (shifts all)

Patterns

  • Two Pointers — sorted arrays, pairs, in-place moves. Converge from both ends or run at different speeds.
  • Sliding Window — contiguous subarray with a constraint (max sum of size k, longest subarray under a limit).
  • Prefix Sum — answer range-sum queries in O(1) after O(n) preprocessing.
  • HashMap — trade space for O(1) lookups (complements, seen-before).

Common questions

  • Two Sum — HashMap of complements, O(n).
  • Best Time to Buy/Sell Stock — track min-so-far, one pass.
  • Product of Array Except Self — prefix & suffix products, no division.
  • Move Zeroes — two pointers, in-place.
  • Merge Intervals — sort by start, then merge overlaps.
Pattern: "contiguous subarray" or "substring" almost always means Sliding Window. "Pair that sums to X" means HashMap or Two Pointers.
Gotcha: Two Pointers needs a sorted array (or a problem where order doesn't matter). If sorting breaks the answer, reach for a HashMap instead.

🎯 Interview questions

How do you spot a sliding-window problem?

It asks about a contiguous range (subarray/substring) optimizing a sum, length, or count under a constraint.

Two Sum in O(n) — how?

Iterate once; for each value check if target - value is already in a HashMap. If yes, return both indices; otherwise store the current value→index.

Product Except Self without division?

Compute prefix products left-to-right, then multiply by suffix products right-to-left in a second pass. O(n) time, O(1) extra (output array aside).

Why is front insertion O(n)?

Every existing element must shift one slot to make room, so cost scales with array length.
