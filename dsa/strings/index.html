Strings DSA

Strings are arrays of characters — the same patterns apply, plus character-frequency tricks. In JS, strings are immutable, so building results means arrays + join or pushing to an output.

Patterns

  • HashMap / frequency count — anagrams, character counts, grouping.
  • Sliding Window — longest/shortest substring with a constraint.
  • Two Pointers — palindromes, reversing, comparing from both ends.

Common questions

  • Valid Anagram — compare character counts, O(n).
  • Group Anagrams — key each word by its sorted form (or count signature).
  • Longest Substring Without Repeating Characters — sliding window + last-seen map.
  • Longest Palindromic Substring — expand around each center, O(n²).

Complexity notes

OperationBig-O
Index a charO(1)
Concatenate in a loopO(n²) — build an array instead
Sort a string of length nO(n log n)
Pattern: "anagram" → frequency map or sorted-key. "longest/shortest substring" → sliding window with a HashMap/Set of what's in the window.
Gotcha: Repeated string concatenation in a loop is O(n²) because each += copies the whole string. Push to an array and join('') once.

🎯 Interview questions

Fastest way to check two strings are anagrams?

If lengths differ, false. Otherwise count chars in one (increment) and subtract for the other; all counts must return to zero. O(n) time, O(1) space for a fixed alphabet.

How does "longest substring without repeats" work?

Slide a right pointer; track each char's last index in a map. When you hit a repeat inside the window, jump the left pointer past its previous occurrence. O(n).

How do you group anagrams?

Map each word to a canonical key (sorted characters or a 26-length count signature) and bucket words sharing the same key.

Why are JS strings immutable a concern?

You can't mutate in place; building output by repeated concatenation is quadratic. Use a char array and join.
