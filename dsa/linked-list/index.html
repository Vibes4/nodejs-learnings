Linked List DSA

A linked list is a chain of nodes, each holding a value and a pointer to the next. No random access, but O(1) insert/delete once you hold the node. Interviews lean on two pointer tricks.

Complexity

OperationBig-O
Access by indexO(n)
Insert / delete at headO(1)
Insert / delete given nodeO(1)
SearchO(n)

Patterns

  • Fast & Slow Pointers (Floyd's) — cycle detection, finding the middle, nth-from-end.
  • Reverse In Place — three pointers (prev, curr, next) walk and flip links.
  • Dummy Head — sentinel node simplifies edge cases when building/merging.

Common questions

  • Reverse Linked List — iterative prev/curr/next, O(n).
  • Detect Cycle — fast moves 2, slow moves 1; they meet inside a cycle.
  • Middle Node — when fast reaches the end, slow is at the middle.
  • Merge Two Sorted Lists — dummy head + splice smaller node each step.
Pattern: Two pointers at different speeds solve "find the middle", "detect a cycle", and "nth from end" — all in one pass, O(1) space.
Gotcha: When reversing, save next before you overwrite curr.next, or you lose the rest of the list.

🎯 Interview questions

How does Floyd's cycle detection work?

A slow pointer (+1) and fast pointer (+2) advance together. If there's a cycle, fast laps slow and they meet; if fast hits null, there's no cycle. O(n) time, O(1) space.

Reverse a linked list iteratively?

Walk with prev=null, curr=head; each step: save next, point curr.next to prev, advance prev and curr. Return prev as the new head.

Find the middle in one pass?

Fast/slow pointers: when fast reaches the end, slow is at the middle.

Linked list vs array — when?

Linked list for frequent insert/delete at known positions and unknown size; array for random access and cache-friendly iteration.
