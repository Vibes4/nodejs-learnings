Binary Search DSA

Binary search halves the search space each step to find a target in O(log n). The non-negotiable precondition: the data must be sorted (or monotonic with respect to your condition).

Complexity

AspectValue
TimeO(log n)
Space (iterative)O(1)
Preconditionsorted / monotonic

The template

  1. lo = 0, hi = n - 1.
  2. While lo <= hi: mid = lo + ((hi - lo) >> 1).
  3. Compare arr[mid] to target; move lo or hi past mid.

Common questions

  • Binary Search — classic find-index-or-(-1).
  • Search Insert Position — where the target would go to keep order sorted.
  • First / Last Occurrence — lower/upper bound when duplicates exist.
  • Search in Rotated Sorted Array — decide which half is sorted, then recurse.
Gotcha: Use mid = lo + Math.floor((hi - lo) / 2) to avoid integer overflow (relevant in fixed-width languages) and watch off-by-one: choosing lo <= hi vs lo < hi changes your loop invariant.
Pattern: "sorted array" + "find/insert/count" → binary search. More broadly, "minimize/maximize X such that a monotonic condition holds" → binary search on the answer.

🎯 Interview questions

What must hold for binary search to work?

The data must be sorted, or the condition you're testing must be monotonic (true-then-false or false-then-true) across the range.

How do you find the first occurrence of a value?

Lower bound: when arr[mid] ≥ target, record it and keep searching left (hi = mid - 1) to find the earliest index.

Why compute mid as lo + (hi - lo)/2?

It avoids integer overflow from lo + hi in fixed-width integer languages; same result, safer.

What is "binary search on the answer"?

When the answer space is monotonic, binary search the candidate value (e.g. minimum capacity, smallest divisor) and test feasibility at each midpoint.
