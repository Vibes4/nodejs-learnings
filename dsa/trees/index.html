Trees DSA

A tree is a hierarchy of nodes with one root and no cycles. The binary tree (≤2 children) dominates interviews. Master the traversals and most tree problems become a one-line recursion.

Traversals

  • DFS (depth-first, via recursion/stack):
    • Preorder — node, left, right. (copy/serialize a tree)
    • Inorder — left, node, right. (sorted output for a BST)
    • Postorder — left, right, node. (delete/free, compute sizes)
  • BFS (breadth-first / level-order, via a queue) — process level by level.

Complexity

OperationBig-O
Traversal (any)O(n)
BST search / insert (balanced)O(log n)
BST search (degenerate / skewed)O(n)

Common questions

  • Maximum Depth — 1 + max(depth(left), depth(right)).
  • Level Order Traversal — BFS, grouping each level.
  • Validate BST — inorder must be strictly increasing (or pass down min/max bounds).
  • Lowest Common Ancestor — recurse; the split point where targets diverge.
Pattern: Inorder traversal of a Binary Search Tree yields values in sorted order — that's the trick behind Validate BST and kth-smallest.
Gotcha: Validate BST is not "left.val < node.val < right.val" locally — every node in the left subtree must be less than the node, so pass down (min, max) bounds.

🎯 Interview questions

When use inorder vs preorder vs postorder?

Inorder → sorted output for a BST. Preorder → serialize/copy (root first). Postorder → free/aggregate children before the parent.

DFS vs BFS on a tree?

DFS (recursion/stack) explores a branch fully before backtracking — natural for depth, paths, subtree aggregates. BFS (queue) explores level by level — natural for shortest depth and level grouping.

How do you validate a BST correctly?

Recurse passing down allowed (low, high) bounds, tightening them as you descend; or check that an inorder traversal is strictly increasing.

Why can a BST degrade to O(n)?

Inserting sorted data makes a skewed, list-like tree. Self-balancing trees (AVL, red-black) keep height O(log n).
