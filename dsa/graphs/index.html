Graphs DSA

A graph is nodes (vertices) connected by edges. Trees and linked lists are special cases. The two must-know traversals — DFS and BFS — plus a visited set to avoid infinite loops solve a huge share of graph problems.

Representations

FormSpaceEdge checkBest for
Adjacency listO(V + E)O(degree)Sparse graphs (most real graphs)
Adjacency matrixO(V²)O(1)Dense graphs, frequent edge lookups

Must know

  • DFS — recursion or explicit stack; go deep, backtrack. Great for connectivity, cycle detection, topological sort.
  • BFS — queue; explore by distance. Gives the shortest path in an unweighted graph.
  • Visited set — mark nodes as you enqueue/visit them, never revisit. Without it, cycles loop forever.

Common questions

  • Number of Islands — flood-fill (DFS/BFS) each unvisited land cell; count components.
  • Clone Graph — DFS/BFS with a map from original node → copy.
Gotcha: Mark a node visited when you enqueue it in BFS, not when you dequeue — otherwise the same node gets added multiple times.
Pattern: A 2D grid is a graph in disguise — each cell connects to its 4 (or 8) neighbors. Islands, flood fill, and shortest-path-in-maze are all grid BFS/DFS.

🎯 Interview questions

DFS vs BFS — which gives shortest path?

BFS, in an unweighted graph, because it expands outward by distance. DFS does not guarantee the shortest path.

Adjacency list vs matrix?

List uses O(V+E) space and is ideal for sparse graphs; matrix uses O(V²) but offers O(1) edge existence checks — good for dense graphs.

Why is traversal O(V + E)?

Each vertex is processed once and each edge is examined a constant number of times.

How do you solve Number of Islands?

Scan the grid; on each unvisited '1', run DFS/BFS to sink the whole connected landmass, and increment a counter once per launch.
