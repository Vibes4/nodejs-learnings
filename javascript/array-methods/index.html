Array & object methods JavaScript

Fluency with the functional array methods and modern object utilities is expected in every coding interview. They make code declarative, and most are non-mutating — key for immutability.

map / filter / reduce

  • map — transform each element 1:1, returns a new array of the same length.
  • filter — keep elements passing a predicate, returns a (possibly shorter) new array.
  • reduce — fold the array into a single accumulated value. Always pass an initial value.
Gotcha: reduce without an initial value uses the first element as the seed and throws on an empty array. Pass the initial value (0, [], {}) to be safe.

find / some / every

  • find — first matching element (or undefined); findIndex for its index.
  • some — true if any element passes (short-circuits).
  • every — true if all elements pass (short-circuits). every on an empty array is true.

Spread & rest

Same ... syntax, opposite jobs. Spread expands an iterable into elements/properties ([...arr], {...obj}, fn(...args)). Rest collects the remainder into an array (function f(...args), const [first, ...rest] = arr).

Note: Spread makes a shallow copy — nested objects are still shared by reference.

Destructuring

Pull values out of arrays/objects by position or key, with defaults and renaming: const { id, name: n = 'anon' } = user, const [a, b] = pair.

Object.keys / values / entries

Iterate objects: Object.keys(o), Object.values(o), Object.entries(o) (array of [key, value] pairs). Object.fromEntries rebuilds an object — handy for transforming maps.

Immutability patterns

Produce new values instead of mutating: [...arr, x] to append, arr.filter(...) to remove, {...obj, key: v} to update a field, arr.map(...) to update items. This is the bedrock of React state and Redux reducers.

Senior tip: Know which methods mutate. push/pop/splice/sort/reverse mutate in place; map/filter/slice/concat/toSorted return new arrays. In immutable code, copy before sorting: [...arr].sort().

🎯 Interview questions

Implement map with reduce.

arr.reduce((acc, x) => [...acc, fn(x)], []) — though pushing into the accumulator is more efficient than spreading each step.

Difference between find and filter?

find returns the first matching element and stops; filter returns all matches as a new array.

Is spread a deep copy?

No, it's shallow — top-level values are copied, but nested objects/arrays remain shared references.

Which array methods mutate?

push, pop, shift, unshift, splice, sort, reverse, fill. The non-mutating counterparts include map, filter, slice, concat, toSorted, toReversed.

How do you update one field immutably?

Spread and override: { ...obj, field: newValue }. For arrays of items, map and replace the target.
