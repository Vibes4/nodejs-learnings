Types & coercion JavaScript

JavaScript has 7 primitive types and one object type. Understanding the difference — and how the engine silently converts between them — is the foundation for everything else.

Primitives vs objects

  • Primitives (immutable, compared by value): string, number, boolean, undefined, null, symbol, bigint.
  • Objects (mutable, compared by reference): plain objects, arrays, functions, dates, etc.

const a = {} === {} is false — two distinct references. But 1 === 1 is true — same value.

== vs ===

=== (strict) compares value and type — no coercion. == (loose) coerces operands to a common type first, which produces surprising results.

Gotcha: == is non-transitive: '' == 0 and 0 == '0' are both true, yet '' == '0' is false. Always use === except for one idiom: x == null to catch both null and undefined.

Falsy values

Exactly 8 values are falsy: false, 0, -0, 0n, '', null, undefined, NaN. Everything else (including [], {}, '0', 'false') is truthy.

NaN & Number checks

  • NaN === NaN is false — the only value not equal to itself.
  • Number.isNaN(x) — true only for the actual NaN value (no coercion). Prefer over global isNaN(), which coerces (isNaN('foo') is true).
  • Number.isInteger(x), Number.isFinite(x) — coercion-free numeric checks.

Object.is

Like === but with two refinements: Object.is(NaN, NaN) is true, and Object.is(0, -0) is false. This is the "SameValue" algorithm.

🎯 Interview questions

Why does typeof null return 'object'?

A bug from the first JS implementation: values were tagged by type bits, and the null pointer (all zeros) shared the object tag. It can never be fixed without breaking the web.

When is == actually acceptable?

The single defensible use is x == null, which is true for both null and undefined and nothing else. Otherwise always ===.

How do you reliably check for NaN?

Number.isNaN(x). The global isNaN() coerces its argument first, so isNaN('abc') wrongly returns true.

Difference between === and Object.is?

Only two cases differ: Object.is treats NaN as equal to NaN, and treats +0 and -0 as distinct.

How do you detect an array?

Array.isArray(x). typeof reports 'object' for arrays, so it can't distinguish them.
