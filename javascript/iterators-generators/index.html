Iterators & generators JavaScript

Iterators define how a collection is traversed; generators are a concise way to write iterators (and to model lazy or infinite sequences). They're what powers for...of, spread, and destructuring.

The iterator protocol

An iterator is any object with a next() method that returns { value, done }. An object is iterable if it has a [Symbol.iterator]() method returning an iterator. Arrays, strings, Maps, and Sets are all built-in iterables.

Symbol.iterator — making an object iterable

Define [Symbol.iterator]() on your object and it instantly works with for...of, spread ([...obj]), Array.from, and array destructuring. This is how you give a custom type "collection" behaviour.

Generator functions

Declared with function*. Calling one returns a generator object (an iterator) without running the body. Each yield pauses execution and emits a value; the next next() resumes right after that yield. The function's local state is preserved between calls — no manual bookkeeping.

Note: A generator is both an iterator and iterable (its [Symbol.iterator] returns itself), so you can drop one straight into for...of. yield* delegates to another iterable.

Lazy sequences

Generators compute values on demand. Nothing runs until you pull the next value, so you can build pipelines (map/filter over a generator) that touch only the elements you actually consume — no intermediate arrays.

Infinite sequences

Because they're lazy, generators can model unbounded streams safely — an infinite while (true) loop with a yield only advances when asked. Take as many values as you need with a counter or a take(n) helper.

Senior tip: Reach for generators for paginated APIs (yield page after page), streaming/large-data processing without buffering everything, and ID/sequence generators. They turn "pull as needed" into a few lines.

🎯 Interview questions

What makes an object iterable?

A [Symbol.iterator]() method that returns an iterator — an object with next() yielding { value, done }. Then for...of, spread, and destructuring all work on it.

What does yield do?

It pauses the generator, emits a value to the caller, and preserves all local state until next() resumes execution right after that yield.

How can a generator be infinite without hanging?

Generators are lazy — the body only advances when next() is called. An infinite loop with yield produces one value per pull, so you control how many you take.

Difference between an iterator and an iterable?

An iterable has [Symbol.iterator]() (a factory for iterators). An iterator has next(). Generators are both — their iterator method returns themselves.

Where are generators genuinely useful?

Lazy/large-data pipelines, infinite or paginated sequences, custom iteration over a data structure, and producing values incrementally without buffering the whole set in memory.
