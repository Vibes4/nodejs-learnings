this & binding JavaScript

this is determined by how a function is called, not where it's defined (arrow functions are the exception). Four rules decide its value, applied in priority order.

The 4 binding rules (highest priority first)

  1. new binding — new Foo() creates a fresh object and binds this to it.
  2. explicit binding — fn.call(obj), fn.apply(obj), or a function returned by fn.bind(obj) force this to obj.
  3. implicit binding — obj.method() binds this to obj (the object left of the dot).
  4. default binding — a plain fn() call: this is the global object (non-strict) or undefined (strict mode / ES modules).
MethodBehaviour
fn.call(ctx, a, b)Invoke now with this = ctx, args listed.
fn.apply(ctx, [a, b])Invoke now, args as an array.
fn.bind(ctx)Return a new function permanently bound to ctx.

Arrow functions capture lexical this

Arrow functions have no this of their own — they inherit it from the enclosing lexical scope at definition time. call/apply/bind cannot change an arrow's this. This makes them ideal for callbacks inside methods (no more const self = this).

Gotcha: Never use an arrow function as an object method if you need this to be that object — it'll capture the outer scope (often module/global) instead.

Losing this in callbacks

Passing a method as a bare reference (setTimeout(obj.method), arr.forEach(obj.method)) strips the implicit binding — it's now a plain call, so this is undefined. Fixes: obj.method.bind(obj), wrap in an arrow () => obj.method(), or define the method as a class field arrow.

Senior tip: The reliable mental model: read the call site, find what's immediately left of (). new > bind/call/apply > obj. > bare call. Arrows opt out entirely and use the surrounding scope.

🎯 Interview questions

What determines this?

The call site, via four rules in priority order: new, explicit (call/apply/bind), implicit (obj.method), then default (undefined in strict mode). Arrow functions ignore all of these and use lexical scope.

Difference between call, apply, and bind?

call and apply invoke immediately (args listed vs. as an array). bind returns a new function with this permanently fixed, to invoke later.

Why is this undefined when I pass obj.method as a callback?

You passed the function value, not the call. The implicit obj. binding only applies at the call site; the callback later invokes it plainly, so this is lost.

Can you rebind an arrow function's this?

No. Arrows capture this lexically at creation; call/apply/bind have no effect on it.

What is this in a standalone function in strict mode?

undefined. In non-strict mode it's the global object (globalThis).
