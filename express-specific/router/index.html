Router (modular routes) Express

express.Router() is a mini-app: a self-contained group of routes + middleware you can mount under a path. It's how you keep large apps organized instead of one giant file.

Pattern

// routes/users.js
const router = express.Router();
router.get('/', listUsers);
router.get('/:id', getUser);
module.exports = router;

// app.js
app.use('/api/users', require('./routes/users'));

Now GET /api/users/42 hits the router's /:id handler. The mount path is stripped before matching inside the router.

Why use it

  • Modularity — one file per resource (users, orders, auth).
  • Router-level middleware — apply auth to a whole router with router.use(auth).
  • Versioning — mount /api/v1 and /api/v2 separately.
  • mergeParams: true — access parent params in nested routers.
Typical structure: routes/ (route definitions) → controllers/ (handlers) → services/ (business logic) → models/ (data). Routers tie the HTTP layer to controllers.

🎯 Interview questions

What is express.Router()?

An isolated, mountable collection of routes and middleware — a "mini Express app" for modularizing a codebase.

How do you apply auth to an entire group of routes?

router.use(authMiddleware) at the top of the router, or pass it when mounting: app.use('/admin', auth, adminRouter).

How do nested routers see parent params?

Create the child router with express.Router({ mergeParams: true }).
