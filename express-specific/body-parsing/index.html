Body parsing Express

HTTP request bodies arrive as a raw stream. Body-parsing middleware reads that stream and populates req.body. Since Express 4.16, the common parsers are built in.

Built-in parsers

  • express.json() — parses application/json → req.body object.
  • express.urlencoded({ extended: true }) — parses HTML form posts (application/x-www-form-urlencoded).
  • express.raw() / express.text() — raw Buffer / plain text.

extended: true vs false

  • false — uses querystring; only string/array values.
  • true — uses qs; supports nested objects (user[name]=x).

multipart / file uploads

The built-ins do not handle multipart/form-data (file uploads). Use multer (or busboy). Multer writes files to disk or memory and adds req.file/req.files.

DoS protection: Always set a body size limit (e.g. express.json({ limit: '1mb' })). Without it, a huge payload can exhaust memory.

🎯 Interview questions

Why is req.body undefined?

No body parser registered, or it's registered after the route, or the client didn't send the matching Content-Type.

How do you handle file uploads?

Use multer for multipart/form-data; the JSON/urlencoded parsers don't handle files.

extended true vs false?

true (qs lib) allows rich nested objects/arrays; false (querystring) is flat. Use true for complex forms.
