Express basics Express

Express is a minimal, unopinionated web framework built on Node's http module. It gives you routing, middleware, and response helpers so you stop writing boilerplate.

The four-line server

const express = require('express');
const app = express();
app.get('/', (req, res) => res.send('Hello'));
app.listen(3000);

Compare that to the raw http module page — Express collapses manual routing, status handling, and response writing into expressive helpers.

The app object

  • app.METHOD(path, handler) — register routes (get, post, put, delete, patch, all).
  • app.use([path,] middleware) — mount middleware.
  • app.set(key, value) / app.get(key) — app settings (e.g. view engine, trust proxy).
  • app.listen(port, cb) — start the server (returns the underlying http.Server).
Install: npm install express. Express 5 is now current (async error handling improved); the concepts here apply to both 4 and 5.

🎯 Interview questions

What is Express, exactly?

A thin routing + middleware layer over Node's http module. It doesn't replace Node; it organizes it.

Is Express opinionated?

No — it's minimal and unopinionated (you choose your DB, structure, auth). Frameworks like NestJS add opinions on top.

What does app.listen return?

The underlying Node http.Server instance — useful for sockets, graceful shutdown, and testing.
