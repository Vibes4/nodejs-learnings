Routing Express

Routing maps an HTTP method + URL pattern to a handler. Express matches routes in the order they're defined — first match wins.

Route patterns

  • Static: app.get('/about', ...)
  • Route params: app.get('/users/:id', ...) → req.params.id
  • Multiple params: /users/:userId/posts/:postId
  • Optional / wildcards: /files/*, regex patterns.
  • Query string: ?sort=asc → req.query.sort (not part of the path).

Chaining & grouping

app.route('/book') lets you chain .get().post().put() for one path. Multiple handlers can be passed to a route (middleware chain) — call next() to advance.

Order matters: Put specific routes before dynamic ones. /users/me must be declared before /users/:id, or :id will capture "me".

route params vs query params

  • Params (/users/42) identify a resource — part of the path.
  • Query (?page=2&sort=asc) filter/modify — optional, after ?.

🎯 Interview questions

How does Express choose which route runs?

It tests routes top-to-bottom in registration order; the first method+path match handles the request (unless it calls next()).

req.params vs req.query vs req.body?

params = path segments (:id); query = after ?; body = request payload (needs a body parser).

Why might /users/:id shadow /users/new?

If :id is declared first, "new" matches as an id. Declare static/specific routes before parameterized ones.
