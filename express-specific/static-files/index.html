Static files Express

express.static serves files (HTML, CSS, JS, images) directly from a folder — no route needed per file. This is built-in middleware.

Usage

app.use(express.static('public'));            // serve ./public at /
app.use('/assets', express.static('public')); // mount at /assets
app.use(express.static(path.join(__dirname, 'public'))); // absolute = safest

A request to /logo.png serves public/logo.png. index.html is served automatically for /.

Useful options

  • maxAge — cache-control max-age for browsers/CDNs.
  • etag, lastModified — conditional requests (304).
  • extensions, index, dotfiles.
Production: Let Nginx/CDN serve static assets, not Node — Node's job is dynamic logic. But for SPAs, Express commonly serves the built front-end bundle.
Always use an absolute path (path.join(__dirname, 'public')). A relative path is resolved against process.cwd(), which breaks when launched from another directory.

🎯 Interview questions

How does Express prevent path traversal in static?

express.static (via the send module) normalizes and blocks paths that escape the root, so ../../etc/passwd is rejected.

Should Node serve static files in production?

Often no — offload to a CDN/reverse proxy for performance. Node serving static is fine for small apps and SPA bundles.
