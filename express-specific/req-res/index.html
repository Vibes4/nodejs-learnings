Request & Response Express

Express extends Node's raw req/res with convenient properties and helpers.

The req object

  • req.params — route params (:id).
  • req.query — parsed query string.
  • req.body — parsed body (needs express.json()/urlencoded()).
  • req.headers / req.get('Header') — request headers.
  • req.cookies — needs cookie-parser.
  • req.ip, req.method, req.path, req.protocol, req.hostname.

The res object

  • res.send(body) — auto Content-Type (string→html, object→json).
  • res.json(obj) — JSON response.
  • res.status(code) — set status (chainable).
  • res.sendStatus(code) — status + matching text.
  • res.redirect(url), res.render(view, data), res.download(file), res.sendFile(path).
  • res.set(header, val) / res.cookie(name, val, opts).
One response only: Calling res.send/res.json twice throws "Cannot set headers after they are sent". Always return after sending.
trust proxy: Behind a load balancer/Nginx, set app.set('trust proxy', true) so req.ip and req.protocol reflect the real client (from X-Forwarded-*).

🎯 Interview questions

res.send vs res.json vs res.end?

res.end = raw Node (no helpers). res.send = sets Content-Type by type. res.json = forces JSON + stringifies.

Why "Cannot set headers after they are sent"?

You sent a response, then tried to send/modify headers again. Usually a missing return before a second res.* call.

How do you get the real client IP behind a proxy?

Enable trust proxy so Express reads X-Forwarded-For; then req.ip is correct.
