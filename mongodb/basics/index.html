MongoDB + Mongoose DB

MongoDB is a document (NoSQL) database storing BSON documents in collections. Mongoose is the dominant Node ODM — adds schemas, validation, middleware, and population on top of the raw driver.

Core concepts

  • Document / Collection ≈ row / table. Flexible schema (but Mongoose enforces one).
  • Schema & Model — define shape, types, validators, indexes; Model = constructor + query interface.
  • _id — 12-byte ObjectId (timestamp + machine + counter), monotonic-ish.

Querying

  • CRUD: create, find, findById, findByIdAndUpdate, deleteOne.
  • Operators: $gt, $in, $or, $regex, projection, sort/skip/limit.
  • Aggregation pipeline — $match → $group → $sort → $lookup for analytics/joins.
  • populate — Mongoose's "join" by reference.

Performance & correctness (senior topics)

  • Indexes — without them queries do COLLSCAN. Create on filtered/sorted fields; understand compound index prefix rules; check with .explain().
  • Embedding vs referencing — embed for "contains/read together", reference for large/shared/unbounded relationships.
  • Transactions — multi-doc ACID requires a replica set; use sessions.
  • Sharding — horizontal scaling by shard key; choose a high-cardinality, even-distribution key.
  • lean() — skip hydrating Mongoose docs for read-heavy endpoints (faster, plain objects).
N+1 in disguise: looping queries (or naive populate over big arrays) kills performance. Batch with $in / aggregation $lookup.

🎯 Interview questions

Embedding vs referencing — how do you decide?

Embed when data is accessed together, bounded, and owned by the parent (one-to-few). Reference for one-to-many/many-to-many, large or independently-queried, or shared data.

How do indexes work and how do you verify usage?

B-tree structures speeding lookups/sorts. Use .explain('executionStats') — look for IXSCAN vs COLLSCAN and docs examined vs returned.

Compound index field order?

Follow ESR: Equality, Sort, Range. A compound index supports queries on its left-to-right prefix.

Does MongoDB support transactions?

Yes, multi-document ACID transactions on replica sets/sharded clusters via sessions. Single-document writes are always atomic.

How do you pick a shard key?

High cardinality, even write distribution, aligned with common query filters — to avoid hot shards and scatter-gather queries.
