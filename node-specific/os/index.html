OS Node Core

The os module exposes operating-system info: CPUs, memory, network, platform. Useful for health checks, sizing worker/cluster counts, and diagnostics.

Key methods

  • os.cpus() — array of logical cores (use .length to size a cluster).
  • os.totalmem() / os.freemem() — RAM in bytes.
  • os.loadavg() — 1/5/15-min load averages (POSIX only).
  • os.platform() ('linux'|'darwin'|'win32'), os.arch(), os.release().
  • os.hostname(), os.uptime(), os.networkInterfaces().
  • os.tmpdir(), os.homedir(), os.EOL (line ending).
Senior tip: Use os.cpus().length to decide how many workers to fork in cluster — typically one per core for CPU-bound work.

🎯 Interview questions

How do you decide cluster worker count?

Usually os.cpus().length (one worker per logical core), but in containers respect CPU limits (cgroups) — os.cpus() may over-report.

process.memoryUsage() vs os.freemem()?

process.memoryUsage() = this Node process (rss, heapUsed). os.freemem() = whole machine. For leak detection watch heapUsed.
