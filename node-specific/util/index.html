Util Node Core

The util module is a grab-bag of helpers. Two are interview-relevant: promisify and util.inspect.

util.promisify

Converts an error-first callback function into one that returns a Promise — so you can await legacy callback APIs.

const readFile = util.promisify(fs.readFile);
const data = await readFile('x.txt', 'utf8');

The inverse is util.callbackify (promise → callback). Note many core modules now ship promise versions natively (fs/promises, timers/promises).

util.inspect

How Node renders objects in console.log. Control depth/colors: util.inspect(obj, { depth: null, colors: true }) — handy for deep nested objects that otherwise print [Object].

Other useful bits

  • util.types.isXxx — reliable runtime type checks (e.g. isAsyncFunction, isDate).
  • util.format — printf-style string formatting (%s, %d, %j).
  • util.deprecate — wrap a function to emit a deprecation warning.

🎯 Interview questions

What does util.promisify require of a function?

It must follow the error-first callback convention (err, result) with the callback as the last argument.

Why does console.log show [Object] sometimes?

Default inspect depth is 2. Use util.inspect(obj,{depth:null}) or console.dir(obj,{depth:null}).
