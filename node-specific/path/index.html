Path Node Core

The path module manipulates file paths in an OS-agnostic way (Windows \ vs POSIX /). Building paths by string concatenation is a bug waiting to happen — always use path.

Key methods

  • path.join(...segments) — join with the correct separator, normalize .. and ..
  • path.resolve(...segments) — produce an absolute path, resolving from right to left (uses cwd if needed).
  • path.basename(p, ext), path.dirname(p), path.extname(p).
  • path.parse(p) → { root, dir, base, name, ext }; path.format() is the inverse.
  • path.sep, path.delimiter (: vs ; for PATH).

join vs resolve

  • join('a','b') → 'a/b' (relative, just concatenation).
  • resolve('a','b') → '/cwd/a/b' (always absolute).
  • A leading / in resolve resets to root: resolve('a','/b') → '/b'.
Security: When building a path from user input, path.resolve + check the result stays inside your base dir — otherwise ../../etc/passwd path traversal is possible.

🎯 Interview questions

join vs resolve?

join concatenates + normalizes (stays relative). resolve produces an absolute path, processing segments right-to-left, falling back to cwd.

How do you prevent path traversal attacks?

Resolve the final path, then verify finalPath.startsWith(baseDir) before touching the filesystem.

How to get __dirname in ESM?

path.dirname(fileURLToPath(import.meta.url)).
