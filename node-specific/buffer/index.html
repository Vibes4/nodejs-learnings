Buffer Node Core

A Buffer is a fixed-length chunk of raw binary memory outside the V8 heap. It's how Node handles bytes: file contents, network packets, crypto, images. Strings are for text; Buffers are for bytes.

Creating buffers

  • Buffer.from(str, encoding) — from a string/array.
  • Buffer.alloc(n) — n zero-filled bytes (safe).
  • Buffer.allocUnsafe(n) — faster, but may contain old memory — fill it before use.

Encodings

'utf8' (default), 'ascii', 'base64', 'hex', 'latin1'. Convert with buf.toString(encoding).

Why allocUnsafe is "unsafe": it reuses memory that may hold sensitive leftover data (passwords, keys). Never send an unfilled allocUnsafe buffer to a client.

Buffers vs strings vs Uint8Array

Buffer is a subclass of Uint8Array, so it works with TypedArray APIs. Multi-byte UTF-8 characters can be split across buffer boundaries when streaming — use string_decoder to handle that.

🎯 Interview questions

Where is Buffer memory allocated?

Outside the V8 heap (raw memory managed by Node), which is why it isn't subject to V8 heap size limits and is efficient for large binary data.

alloc vs allocUnsafe?

alloc zero-fills (safe, slightly slower). allocUnsafe skips zeroing (faster) but may expose old memory — must fill before exposing.

How can string concatenation of stream chunks corrupt data?

A multi-byte UTF-8 char split across two chunks decodes wrong. Concatenate Buffers first, or use StringDecoder.
